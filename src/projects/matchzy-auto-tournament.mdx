MatchZy Auto Tournament is a web platform I built to automate Counter-Strike 2 tournaments from bracket creation all the way to final scores. The goal was to remove as much manual operator work as possible by connecting tournament structure, server orchestration, veto flow, match lifecycle, and live results into one system.

The platform supports tournament formats like single and double elimination, Swiss, round robin, and shuffle, along with browser-based map veto, automatic server allocation, live score tracking, bracket progression, demo handling, and public match pages. A big part of the value in this project was designing it around real tournament operations, where reliability, clear admin tooling, and low manual overhead matter just as much as the features themselves.

From a technical perspective, this project gave me experience building a full operational platform rather than just a single app. I worked on real-time systems, backend workflows, infrastructure integration, API design, and the operational side of coordinating CS2 servers with tournament state. It also reflects how I think about product engineering: building tools that solve an entire workflow, not just one isolated piece of it.

This project gave me strong experience in systems design, automation, full-stack product development, and building software for real-world operations under live conditions.

**Tech:** TypeScript, Docker, WebSockets, API design, CS2 server orchestration, MatchZy Enhanced, tournament automation, real-time state management
