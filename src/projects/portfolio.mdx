This portfolio is the site I built to present my work in a way that feels closer to product storytelling than a traditional résumé. Instead of listing projects as static entries, I designed it as a technical showcase where each project can explain what it does, what I built, and what kind of engineering work it involved.

The site itself reflects how I like to build front-end products: clean structure, reusable components, motion used with restraint, and content that is easy to extend over time. I built it with React, TypeScript, Vite, Tailwind, and MDX, which made it possible to combine a fast UI with a flexible writing workflow for project pages and long-form content.

What makes this project valuable to me is that it is both a product and a system. It is the public-facing representation of my work, but it also gave me room to design content architecture, reusable presentation patterns, and a deployment setup that stays simple and maintainable. It shows how I think about developer experience just as much as user experience.

This project gave me hands-on experience with front-end architecture, content-driven UI design, reusable component systems, and building a polished production-ready web application for my own work.

**Tech:** React, TypeScript, Vite, Tailwind CSS, Motion, MDX, React Router, Bun, Docker, Nginx
