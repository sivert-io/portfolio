## What problem does it solve?

Managing Counter-Strike 2 servers manually is painful.  
Installing plugins, updating servers, configuring RCON, and scaling multiple servers becomes tedious quickly.

CS2 Server Manager was built to automate that process.

It allows tournament organizers and communities to deploy and manage multiple CS2 servers from a single tool.

## Why I built it

While building tournament infrastructure I constantly had to:

- spin up multiple match servers
- configure plugins
- update game servers safely
- manage RCON credentials
- avoid breaking configurations during updates

This project started as a personal automation tool and evolved into a full CLI/TUI management system.

## Architecture

Main stack:

- **Go**
- **Bubble Tea** (terminal UI)
- **SteamCMD**
- **CounterStrikeSharp plugins**
- **MatchZy integration**

High-level flow:

```

CLI / TUI
↓
Server manager core
↓
SteamCMD installer
↓
Plugin & config manager
↓
CS2 servers

```

## Key Features

- Multi-server deployment
- Safe plugin updates
- MatchZy tournament integration
- Persistent configuration overrides
- Live terminal UI management
- tmux integration for server processes
